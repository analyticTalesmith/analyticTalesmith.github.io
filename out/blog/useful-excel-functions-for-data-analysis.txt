<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-a507ada" id="some-definitions" data-block-id="a507ada"><h2 class="stk-block-heading__text">Some Definitions</h2></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-8c2a76d" data-block-id="8c2a76d"><p class="stk-block-text__text">Before we get into the list of functions, following are some words you should be familiar with to help understand the various functions:</p></div>



<ul class="wp-block-list">
<li class=""><strong>Argument </strong>— The input that you provide when typing in a function which gets passed into a function&#8217;s parameter.</li>



<li class=""><strong>Criteria </strong>— Often used when functions evaluate conditionally over a range of cells, the criteria is an argument that you specify to say &#8220;if the cell in that range meets this criteria, then use it in the function.&#8221; Note that you can use an array constant if you want multiple conditions included in your criteria.</li>



<li class=""><strong>Logical </strong>— A function that ultimately evaluates to either TRUE or FALSE. </li>



<li class=""><strong>Parameter </strong>— The name given to a variable in a function that references a piece of the provided input.</li>
</ul>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-744505d" id="quick-excel-tips" data-block-id="744505d"><h2 class="stk-block-heading__text">Quick Excel Tips</h2></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-b2a3e3f" data-block-id="b2a3e3f"><p class="stk-block-text__text">Want more information about a function when typing it into Excel? Open the Excel Help menu and search for the function. Alternatively, start typing the function in a cell, and when you see the tooltip come up that tries to autocomplete the function you are typing, double click the function name, and then mouse your cursor over the function name. You will see a hyperlink to open the function documentation in the Excel Help menu. </p></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-5adc32f" data-block-id="5adc32f"><p class="stk-block-text__text">Want a graphical user interface to make typing your functions a little easier? Press Ctrl A after you have typed &#8220;=&#8221; and the full name of the function in a cell. This will either bring up the function arguments window <em>or </em>will first bring up a window to select which version of the function you want to use and <em>then </em>bring up the function arguments window.</p></div>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-0084988" id="conditional-functions" data-block-id="0084988"><h2 class="stk-block-heading__text">Conditional Functions</h2></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-31e7f7c" data-block-id="31e7f7c"><p class="stk-block-text__text">By far <em>the </em>most useful set of functions I have found over my years are conditional functions. Generally speaking, they let you perform various functions only if the condition(s) you specify is met.</p></div>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-ff04110" id="if" data-block-id="ff04110"><h3 class="stk-block-heading__text">IF</h3></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=IF(logical_test, [value_if_true], [value_if_false])" style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=IF(logical_test, [value_if_true], [value_if_false])</span></span></code></pre></div>



<p class="">IF is perhaps <em>the </em>function I use the most frequently in Excel. Given a logical test, you can then specify what you want to have happen when the test is TRUE. Or if you include an argument for value_if_false, what you want to happen depending on whether the test is TRUE or FALSE.</p>



<p class="">IF is especially useful once you learn how to nest it. Like other syntaxes, you can include multiple layers of nesting to have branching logic to determine what happens in your cell. Admittedly, I don&#8217;t often have nested IFs that have more than 2 or 3 layers, but if you find yourself needing to create nested IFs, simply pass another IF function as an argument into the appropriate &#8220;value_if&#8221; parameter like shown below.</p>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=IF(logical_test,
  IF(logical_test, [value_if_true_and_true], [value_if_true_and_false]),
  IF(logical_test, [value_if_false_and_true], [value_if_false_and_false]))" style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=IF(logical_test,</span></span>
<span class="line"><span style="color: #d8dee9ff">  IF(logical_test, [value_if_true_and_true], [value_if_true_and_false]),</span></span>
<span class="line"><span style="color: #d8dee9ff">  IF(logical_test, [value_if_false_and_true], [value_if_false_and_false]))</span></span></code></pre></div>



<p class="">In this example, the nested IF functions will evaluate to one of four &#8220;value_if&#8221; functions depending on whether the first logical test is TRUE or FALSE and then whether the logical test within that branch is TRUE or FALSE.</p>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-f09d189" id="ifs" data-block-id="f09d189"><h3 class="stk-block-heading__text">IFS</h3></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-1a286b5" data-block-id="1a286b5"><p class="stk-block-text__text">You might find yourself wanting to have an IF function in Excel that works like a switch statement or an if/elseif/else block in other languages. IFS is a great option to achieve this in Excel.</p></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=IFS(logical_test1, value_if_true1, [logical_test2, value_if_true] ...)" style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=IFS(logical_test1, value_if_true1, [logical_test2, value_if_true] ...)</span></span></code></pre></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-b36db2c" data-block-id="b36db2c"><p class="stk-block-text__text">IFS requires at least one logical and one value if that test is TRUE, but you can optionally include more logical tests. The function will check if the first condition is TRUE; if so, evaluate and return whatever argument you pass into value_if_true1. If the condition is FALSE and you have a second condition, then it checks the second condition the same way. The function will move through each of the logical/value pairs you pass as an argument until it either finds a level that evaluates to TRUE or all logical tests are FALSE (at which point you will see a #N/A error).</p></div>

<p class="">AND, OR, and XOR all work by checking a list of conditions. AND will return TRUE if every condition is TRUE; OR will return TRUE if at least one condition returns TRUE; XOR will return TRUE if an odd number of the logicals return TRUE.</p>



<p class="">Note that with XOR, that means that an XOR function with 3 logicals can and will return TRUE if all 3 are TRUE, and that for XOR function with 3 or more logical tests, a final value of TRUE does <em>not </em>necessarily mean that <em>only </em>one logical evaluated to TRUE.</p>



<p class="">NOT isn&#8217;t necessary strictly speaking, <em>but </em>it can be useful to make complex functions a little more readable. It simply inverts TRUE to FALSE and FALSE to TRUE. <div class="wp-block-stackable-text stk-block-text stk-block stk-167cb47" data-block-id="167cb47"><p class="stk-block-text__text">Want to get the minimum and maximum values of a subset of data based on criteria? Lo and behold; there are premade functions for that too.</p></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=MINIFS(min_range, criteria_range1, criteria1, [criteria_range2, criteria2] ...) <div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-5bcdc34" id="iferror" data-block-id="5bcdc34"><h3 class="stk-block-heading__text">IFERROR</h3></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-b95da90" data-block-id="b95da90"><p class="stk-block-text__text">I would be lying if I said I used IFERROR <em>often</em>, but I <em>do </em>still use it, especially when I&#8217;m working on a workbook that I know other people will be opening and potentially trying to edit.</p></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=IFERROR(value, value_if_error)" style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=IFERROR(value, value_if_error)</span></span></code></pre></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-40497a8" data-block-id="40497a8"><p class="stk-block-text__text">Simply put, this function will return whatever argument you pass into value if it detects an error. My most common use of this function is to <em>try </em>to include some troubleshooting guidance to users by predicting what error they might make and then passing a string to display to guide the debugging.</p></div>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-fd3f064" id="lookup-functions" data-block-id="fd3f064"><h2 class="stk-block-heading__text">Lookup Functions</h2></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-fc16cea" data-block-id="fc16cea"><p class="stk-block-text__text">These functions boil down to one simple purpose: you have a value, and you need to find a related value in another table.</p></div>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-4d2dba6" id="xlookup" data-block-id="4d2dba6"><h3 class="stk-block-heading__text">XLOOKUP</h3></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-04d12cd" data-block-id="04d12cd"><p class="stk-block-text__text">In the past, I might include HLOOKUP and VLOOKUP here, but I have yet to find any reason to use them over XLOOKUP, which has the flexibility of both other types of lookups in one, without the limitation of left-to-right search only. </p></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])  " style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])  </span></span></code></pre></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-723f150" data-block-id="723f150"><p class="stk-block-text__text">The function works by looking through the array or range in lookup_array (a column or row) for the lookup_value, and then if that value is found, returning the value from the same row or column in return_array.</p></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-b25a599" data-block-id="b25a599"><p class="stk-block-text__text">Optionally, you can also specify what to return instead of a #N/A error if the text is not found by passing an argument into if_not_found.</p></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-1d60b6a" data-block-id="1d60b6a"><p class="stk-block-text__text">Match_mode lets you specify an exact match (0, the default) where if_not_found or #N/A is returned if the match isn&#8217;t found, an exact match where the next smaller (-1) or next larger (1) item is returned, or a wildcard match (2).</p></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-58c5268" data-block-id="58c5268"><p class="stk-block-text__text">Search_mode lets you specify whether to do a forward search from the first item (1, the default), a reverse search from the last item (-1) or a binary search if your data are already sorted in ascending (2) or descending (-2) order.</p></div>



<div class="wp-block-stackable-heading stk-block-heading stk-block-heading--v2 stk-block stk-6b49e5d" id="index-and-xmatch" data-block-id="6b49e5d"><h3 class="stk-block-heading__text">INDEX and XMATCH</h3></div>



<div class="wp-block-stackable-text stk-block-text stk-block stk-06e0681" data-block-id="06e0681"><p class="stk-block-text__text">I generally do not run into many cases where XLOOKUP isn&#8217;t sufficient for my lookup needs, but if you need to have more precise control over your lookup and returns (e.g., perhaps you need to perform a two-way/matrix lookup), INDEX and XMATCH are a great option. Note that INDEX has an array form and a reference form.</p></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=XMATCH(lookup_value, lookup_array, [match_mode], [search_mode])   " style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=XMATCH(lookup_value, lookup_array, [match_mode], [search_mode])   </span></span></code></pre></div>



<div class="wp-block-kevinbatdorf-code-block-pro" data-code-block-pro-font-family="Code-Pro-JetBrains-Mono" style="font-size:.875rem;font-family:Code-Pro-JetBrains-Mono,ui-monospace,SFMono-Regular,Menlo,Monaco,Consolas,monospace;line-height:1.25rem;--cbp-tab-width:2;tab-size:var(--cbp-tab-width, 2)"><span role="button" tabindex="0" data-code="=INDEX(array, row_num, [column_num])" style="color:#d8dee9ff;display:none" aria-label="Copy" class="code-block-pro-copy-button"><svg xmlns="http://www.w3.org/2000/svg" style="width:24px;height:24px" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path class="with-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2m-6 9l2 2 4-4"></path><path class="without-check" stroke-linecap="round" stroke-linejoin="round" d="M9 5H7a2 2 0 00-2 2v12a2 2 0 002 2h10a2 2 0 002-2V7a2 2 0 00-2-2h-2M9 5a2 2 0 002 2h2a2 2 0 002-2M9 5a2 2 0 012-2h2a2 2 0 012 2"></path></svg></span><pre class="shiki nord" style="background-color: #2e3440ff" tabindex="0"><code><span class="line"><span style="color: #d8dee9ff">=INDEX(array, row_num, [column_num])</span></span></code></pre></div>

<div class="wp-block-stackable-text stk-block-text stk-block stk-d9eef14" data-block-id="d9eef14"><p class="stk-block-text__text">Generally speaking, regardless of your specific needs for using these functions, you will use XMATCH (or MATCH) to find the relative location of a lookup_value in a lookup_array, and nest that within an INDEX function to return a related value.</p></div>
